# Innovation Lab

**What is this?** Vita's permanent idea incubator — the single source of truth for every product idea, regardless of whether it ships next week or in five years.

**Why does it exist?** Good ideas are cheap to lose and expensive to recreate. The Innovation Lab captures every idea — feature, AI capability, UI/UX notion, monetization angle, growth tactic, integration, experiment, or future product — refines it into a real proposal, and preserves it permanently. **Ideas are never deleted.** If an idea changes, the note is updated, not replaced.

> **The Innovation Lab is not the development roadmap.** The [[Roadmap]] is what we've committed to build and when. The Innovation Lab is everything we might build, at every stage of maturity, whether or not it's scheduled. An idea enters the Roadmap only when it's promoted deliberately — see [[Roadmap|Idea Promotion]] below.

## The idea lifecycle

Every idea moves through the same six stages. Status lives at the top of each idea note.

💭 **Brainstorm** → 📝 **Defined** → 📋 **Planned** → 🏗 **In Development** → 🧪 **Testing** → ✅ **Released**

| Stage | Means |
|---|---|
| 💭 Brainstorm | Captured. Rough is fine — the core concept is extracted and given a home. |
| 📝 Defined | Problem, solution, and value are articulated. Not yet scoped or scheduled. |
| 📋 Planned | Scoped into a future sprint or explicitly prioritized by the founders. |
| 🏗 In Development | Actively being built as a slice ([[Claude Workflow]]). |
| 🧪 Testing | Built, in founder review or user testing. |
| ✅ Released | Shipped. Moved to reference under [[Implemented Features]]. |

Ideas never regress out of the system and never get deleted — a declined or shelved idea stays at its last real stage with a note explaining why, so it can be revisited without re-litigating from scratch.

## Every idea note includes

Title · Status · Category · Priority (Low/Medium/High/Critical) · Estimated Difficulty (Low/Medium/High/Very High) · Recommended Tier (Free/Pro/Atlas/Elite/Internal) · Problem · Solution · User Experience · Why Users Would Love It · Business Value · Dependencies · Future Enhancements · Related Ideas · Tags · **Implementation Readiness** (founders, 2026-07-09).

**Implementation Readiness** is the closing section on every note — a portfolio-review snapshot distinct from the Status field above it, so a reader months later can tell instantly whether an idea is still just a concept or nearly ready to build: Current Status (🟢 Concept Complete / 🟡 Needs Refinement / 🔴 Blocked), Next Step Required, Estimated Sprint, condensed Dependencies, and a Confidence score out of 10.

Template: [[Idea Template]].

> **Tier vocabulary note:** Free/Pro/Atlas/Elite/Internal are working tier names used for idea triage. No pricing or tier structure has actually been decided — see [[Business Model & Pricing]].

## Reports

Generated on request from the current set of ideas — not live queries (no Dataview in this vault), so treat each as a dated snapshot to regenerate when asked:

[[Highest Impact Ideas]] · [[Quick Wins]] · [[Atlas Features Report]] · [[Premium Opportunities]] · [[AI Features Report]] · [[Low Effort High Impact]] · [[Ready for Development]] · [[Recently Added]]

These help *inform* sprint planning. They never replace the [[Roadmap]].

## How ideas get in

Whenever a founder provides an idea — however rough — Claude: extracts the core concept, writes it up against the full template, expands it where appropriate without changing the intent, files it in the right category, links related ideas, and never criticizes an idea for being difficult or removes one for being unrealistic today. Everything is preserved so it can be revisited.

## Promoting an idea to the Roadmap

An idea graduates from the Lab to the [[Roadmap]] only when the founders commit to building it: it passes the **VITA Promise** ([[Product Philosophy]]) and the **"Build for the Next 10 Years"** test ([[Core Principles]]), gets logged in the [[Decision Log]], and its status moves to 📋 Planned. The idea note stays in the Lab — the Roadmap links back to it rather than duplicating it.
