# Atlas Capabilities

**What is this?** How Atlas's future capabilities relate to and build on each other — the sequencing logic, not the detailed proposal for each one. Full proposals (problem, solution, UX, business value) now live in the [[Innovation Lab]] under [[Atlas Features]]; this page is the "how they compound" story.

**Why does it exist?** These are founder future priorities that shape today's data decisions. One page keeps the sequencing coherent even as individual capabilities get their own detailed Lab notes.

*Status: nothing implemented; no AI code in the repo (verified 2026-07-06). ⚠️ The "post-V1 only" framing below is superseded — the official [[Roadmap]] (2026-07-09) commits most of these to Sprint 4–6. See the [[Decision Log]] for the reversal of the prior "Atlas V1 is a polished placeholder only" decision.*

## Sequencing — official roadmap order (2026-07-09)

**Sprint 4 (Atlas): Meal Planning and Workout Planning together → Sprint 5 (Health): Health Age logic → Sprint 6 (Premium): Voice Atlas.** This mostly matches the sequencing this page previously recommended, with one difference: the original recommendation sequenced Workout Generation *after* Health Age logic, gated on the tracking-module question resolving first; the official roadmap instead commits Meal Planning and Workout Planning to the same sprint. The tracking-module scope question itself is still not fully resolved by that commitment — see [[Workout Generation]].

Advanced Coaching (proactive check-ins) isn't named in any sprint yet — it's adjacent to Sprint 4's Recommendations slice and Sprint 6's Smart Notifications slice, but hasn't been promoted.

## Dependencies

Real user data (Sprints 1–5) before any of this is buildable — Atlas without data is a generic chatbot, which is exactly what the product must not ship ([[Future AI]]).
